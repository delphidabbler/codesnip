File Format Documentation

Saved Files

Introduction

CodeSnip saves external files in three different ways:

  1. By saving snippet information to file from the File | Save Snippet Information menu option.
  2. By saving snippets to file from the File | Save Snippet menu option.
  3. By saving units to file from the File | Save Unit menu option.

In the first case the snippet is always saved in rich text format.

In the second two cases the following file types can be chosen by the user:

  • Plain text.
  • Pascal source files (either .inc or .pas files).
  • HTML 5 files.
  • XHTML files.
  • Rich text files.

There is no specific file format for these files, except that HTML 5, XHTML and RTF files conform to published specifications.

Encoding

In the first case the RTF is always saved in ASCII format.

In the 2nd and 3rd cases the encodings used depend on the file type and user choice. Different file types have different encoding choices, as follows:

Plain text
  • ANSI (system default code page)
  • UTF-8
  • Unicode little endian (UTF16-LE)
  • Unicode big endian (UTF16-BE)
Pascal source files
  • ANSI (system default code page)
  • UTF-8
HTML 5 files
  • UTF-8
XHTML files
  • UTF-8
Rich text files (RTF)
  • ANSI (system default code page). ASCII format is actually used.

Files with UTF-16LE, UTF-16BE and UTF-8 encodings are saved with byte order marks.
