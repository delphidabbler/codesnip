Import Wizard

This wizard is used to import user defined snippets from a file exported by another user. It is accessed from the Snippets | Import Snippets menu option.

The different pages of the wizard are discussed below. You can cancel the import any time the Cancel button is enabled.

First page

This is simply an introductory page that describes the wizard. Click the Next button to display the Choose import file page.

"Choose import file" page

This page is used to specify the file you want to import. Enter the file name in the Open import file edit box.

Alternatively, the required file can be chosen from the standard Windows Import File dialogue box. Click the ellipsis button to the right of the edit control to display it. The name of the selected file is copied to the Open import file edit box.

Files exported from CodeSnip usually have a .csexp extension.

Once a suitable file has been specified click the Next button to load the file and move to the next page of the wizard. The database is not updated at this stage, so if you made a mistake you can return to this page and choose another file.

"Edit import and update database" page

This page lets you rename or reject any of the imported snippets before adding them to the database.

Each snippet in the import file is listed in a grid. The columns are:

  • Snippet Name
    This is the name of the snippet in the import file, i.e. the name of the snippet in the file exporter's database. Clear the check box next to the snippet if you don't want to import it.
  • Import Using Name
    This is the name that will be used for the snippet when it is imported. If there is already a snippet in your database with the same name as that given in the Snippet Name column, the entry in this column will have been altered to make it unique in your database. Any names in this column can be edited (see below).
  • Action
    This column describes what will happen to this snippet when the database is updated. It will either be "Import" if the snippet will be imported (check box checked) or "Skip" if the snippet will not be imported (check box cleared).

Selecting one of the rows in the grid copies the snippet name from the Import Using Name column into the Selected Snippet edit control. You can rename the snippet by entering a new name in the edit control then clicking the Rename button. The new name is validated before being displayed in the grid. You can't use the name of a snippet that is already in your user database or is the name of another snippet in the import list.

When you are ready to perform the import click the Update button to update the database. There will be a pause while the snippets are imported then the final wizard page will be displayed.

The newly imported snippets will be located in the "Imported Snippets" database category ready for you to move or edit them.

Note: Once the Update button has been clicked you can't cancel or undo the update.

If no snippets have been selected an error message will be displayed when the Update button is clicked.

"Import complete" page

This page is displayed once the database update has completed. It simply lists the names of the snippets that have been imported.

Close the wizard by clicking the Finish button. Note that the Back button is disabled because you can't undo the update.
