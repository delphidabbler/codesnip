Move User Database Dialogue Box

Note: This dialogue box is not available in the portable edition of CodeSnip.

The purpose of this dialogue box is to enable the user to move the user defined snippets database from its default location to a new directory. You may wish to do this to enable the database to be easily backed up. For example you could move the database to a sub-directory of a Dropbox or GoogleDrive directory.

You have the option to restore a previously moved database to its default location. Both options are described below.

Important: You a strongly advised to make a backup of the user defined snippets database before moving it so that it can be easily restored if there is a problem. Use the Database | Backup User Database menu option to do this.

Move the database to a new directory

To move the database enter the full path of the new directory in the edit box. Instead of typing the path you can choose it from a standard Browse For Folder dialogue box by clicking the ellipsis button to the right of the edit control.

The directory where the database must meet the following conditions:

  • it directory must be empty;
  • it must not be a sub-directory of the current database location;
  • it must be on a drive on the current machine (removable drives are permitted, but not recommended.

Once a suitable path has been entered, click the large Move button. You will be prompted to confirm this action and then the database files will be copied. An error message will be displayed if the database can't be copied.

The new location of the database will be recorded in the per user config file and it is important that this file is not modified or deleted otherwise the database will not be found.

Restore database to default directory

The database can be moved back to its default directory simply by clicking the Restore Default Path button. If the default directory is not empty this action will fail and an error message will be displayed.

Yhe Restore Default Path button will be disabled if database is already in its default location.
