Move User Database Dialogue Box

Note: This dialogue box is not available in the portable edition of CodeSnip.

The purpose of this dialogue box is to enable the user to move the user defined snippets database from its default location to a new directory. You may wish to do this to enable the database to be easily backed up. For example you could move the database to a sub-directory of a Dropbox or GoogleDrive directory.

Important: You a strongly advised to make a backup of the user defined snippets database before moving it so that it can be easily restored if there is a problem. Use the Database | Backup User Database menu option to do this.

To move the database enter the full path of the new directory in the New database path edit box. Alternatively you can use a standard Browser For Folder dialogue box to select the required directory. To do that click the ellipsis button to the right of the edit control, select the directory in the resultant dialogue box and click OK. The chosen directory will be displayed in the edit box.

The directory where the database is to be copied must be empty and must be on a valid drive.

Once a suitable path has been entered click the large Move Database button. You will be prompted to confirm this action and then the database files will be copied.

The new location of the database will be recorded in the per user config file and it is important that this file is not modified or deleted otherwise the database will not be found.

The database can be moved back to its default location by clicking the Restore Default Path button. This copies the default directory into the New database path edit control. Clicking the Move Database button will now move the database back to its default directory.

Note that the Move Database button will be disabled if the directory entered in the New database path edit control is the current database path or if the edit control is empty.
