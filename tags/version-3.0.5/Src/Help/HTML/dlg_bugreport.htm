Bug Report Wizard

This dialog box is used to report bugs online and can be initiated in two different ways:

  1. By the user selecting the Tools | Report Bug Online menu option. The dialog box is captioned "Bug Report"
  2. Automatically when CodeSnip detects a likely bug. The dialog box is captioned "Bug!"

In either case the dialog box takes the form of a multi-page wizard that gathers the necessary data to report the bug. You move through the pages by clicking the Next button and can return to a previous page using the Back button. The Cancel button closes the wizard without reporting the bug.

First page

This page differs depending on how the dialog was initiated:

  • When the dialog is intiated by the user the page is titled Bug report wizard and it simply displays a brief introductory message.
  • When displayed because a bug has been detected the page is titled CodeSnip has encountered an error. A description of the unexpected program event that was detected is displayed on the page along with some introductory text.

Second page: Step 1: About the bug

This page is where a description of the bug is entered. Please provide as much detail as possible about the effect of the bug and what you were doing at the time it was noticed. Whether this description is compulsory or optional depends on how the dialog was intiated:

  • For a user initiated dialog the description is required. An error message will be displayed if no description is entered.
  • When CodeSnip displays the dialog automatically the program already has some information about the bug (i.e. that which was displayed on the first page). Therefore, although additional information is helpful, the description is optional. You will be warned if you don't enter a description but you can proceed without entering anything.

Third page: Step 2: About you

On this page you should enter your email address if you would like a reply from the author concerning the bug. This is optional. If you are concerened about how your email address will be used click the privacy statement link that appears on the page. CodeSnip does some basic checks on the validity of the email address before proceeding.

Details of your computer's operating system are also displayed. This information will be included in the bug report.

Fourth page: Step 3: Send the bug report

The contents of the bug report are displayed on this page. This is done so you are aware exactly what information is to be sent. This information includes:

  • The bug report version.
  • CodeSnip's DelphiDabbler ID, name and version numbers.
  • Details of your operating system.
  • A description of event that triggered the dialog box (automatic reports only).
  • Your description of the bug.
  • You email address, if provided.

If you are happy to send the information click the Submit button which will have replaced the Next button. Note: please ensure you are connected to the internet before submitting the report.

The program will now attempt to connect to www.delphidabbler.com and send the report to the website's bug report service. If the report is sent successfully the "Finished" page will be displayed. Should an error be detected while sending the report an error message describing the problem will be displayed.

Fifth page: Finished

This page is displayed when a bug report has been sent successfully. It simply displays confirmation that the report was sent. You can close the dialog box by clicking the Close button that will have replaced the previous page's Submit button.
